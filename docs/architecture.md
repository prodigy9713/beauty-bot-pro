# Architecture

## Общая идея

Проект разделен на небольшие понятные части, чтобы его было легко поддерживать, дорабатывать и продавать как шаблон.

## Слои проекта

- `handlers` принимают сообщения и callback-кнопки Telegram;
- `keyboards` создают reply и inline клавиатуры;
- `services` содержат бизнес-логику и работу с SQLite;
- `states` описывают пошаговые сценарии через FSM;
- `database.py` создает таблицы и стартовые данные;
- `config.py` читает настройки из `.env`;
- `main.py` запускает бота и подключает роутеры.

## Таблицы SQLite

- `services` хранит услуги;
- `time_slots` хранит свободные и занятые окна;
- `appointments` хранит записи клиентов;
- `reviews` хранит отзывы;
- `settings` хранит редактируемые тексты бота.

## Текущий подход

На старте выбран `SQLite`, потому что:

- легко запускать локально;
- не нужен отдельный сервер базы данных;
- удобно для MVP и демонстраций;
- потом можно перейти на PostgreSQL без полной перестройки логики.
