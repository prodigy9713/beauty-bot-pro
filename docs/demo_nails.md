# Demo Nails

## Что важно понять

Если у вас уже есть старая база `beauty_bot.db`, новые стартовые услуги и тексты из кода автоматически не перезапишут старые данные.

Для чистой демо-версии лучше сделать так:

1. Создать отдельного тестового бота в BotFather.
2. В `.env` для тестового бота указать другой `BOT_TOKEN`.
3. Указать отдельную базу, например:

```env
DATABASE_PATH=beauty_demo_nails.db
```

4. Запустить бота с новой пустой базой. Тогда демо-данные под маникюр и педикюр подтянутся автоматически.

## Что будет в демо после первого запуска

- маникюр с покрытием гель-лак;
- укрепление + маникюр;
- педикюр с покрытием;
- снятие чужого покрытия;
- тексты под nail-нишу;
- отзывы под маникюр и педикюр.

## Сценарий для скриншотов

1. `/start`
2. Открыть `💅 Услуги`
3. Открыть карточку `Маникюр с покрытием гель-лак`
4. Открыть `💰 Прайс`
5. Открыть `⭐ Отзывы`
6. Открыть `📅 Записаться`
7. Показать выбор даты и времени
8. В админке показать добавление свободного окна

## Какие 6 скриншотов сделать

- главный экран после `/start`;
- список услуг;
- карточка одной услуги;
- прайс;
- отзывы;
- запись на свободное время.

## Подпись к демо
